异步编程概念#

异步编程是一种编程模式，它允许程序在等待某些操作完成时继续执行其他代码。在 C# 中，异步编程可以通过使用异步方法、任务和协程等来实现。

传统多线程开发过于麻烦； C# 关键字：async、await； async、await 不等于“多线程”。

异步方法#

异步方法是使用 async 和 await 关键字定义的。它们允许您编写看起来像同步代码的异步代码，并且可以在等待操作完成时释放线程。例如：

public async Task MyAsyncMethod()  {
	await DoSomethingAsync();
	// 其他代码可以在等待DoSomethingAsync完成时继续执行
}

异步方法：

  1. Async 关键字修饰的方法
  2. 返回值一般是 Task<T>，即使方法没有返回值，也最好把返回值声明为非泛型的 Task
  3. 异步方法名字一般以 Async 结尾
  4. 调用泛型方法时，一般在方法前加上 await 关键字，这样拿到的返回值就是泛型指定的 T 类型
  5. 异步方法的“传染性”：一个方法中如果有 await 调用，则这个方法也必须修饰为 async

任务#

任务是表示异步操作的类，可以使用 Task 类或 Task<TResult>类来表示。Task 类表示异步操作没有返回值，而 Task<TResult>类表示异步操作返回一个值。例如：

Task<string> task = Task.Run(() => GetStringAsync());
string result = await task; // 等待任务完成并获取结果

协程#

协程是一种轻量级的控制流工具，可以在不需要使用线程的情况下实现异步编程。协程可以让您在等待异步操作完成时暂停和恢复代码执行。例如：

async void MyCoroutine()  {
	await DoSomethingAsync(); // 等待异步操作完成
	// 其他代码可以在等待DoSomethingAsync完成时继续执行
}

注意事项#

在使用异步编程时，需要注意以下几点：

  • 避免过度异步化：过多的异步操作可能会导致代码难以理解和维护。
  • 正确处理异常：在异步代码中，异常处理非常重要，需要确保正确捕获和处理异常。
  • 避免死锁：在使用 await 关键字时，需要确保不会出现死锁的情况。
